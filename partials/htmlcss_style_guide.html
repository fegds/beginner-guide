命名规范

CSS的命名我们使用“模块说明”的一种命名规则，无论ID还是CLASS都使用小写单词，多个单词间用横杠“-”连接，比如：portal-ui。

这样做是为了能清晰的了解某一模块的用处，而不是使用一些无意义的名字。但是在特定情况内的命名，无需做到“结构路径”全部写上，比如： 

<div class="porta-ui-win">
	<div class="portal-ui-win-title"></div>
</div>

上面的这种方式如果在层级比较多的情况下又会变得很冗余，虽然是按照规则来的，但是不推荐这样使用，因为portal-ui-win-title的样式是只适用于portal-ui-win里面，所以我们可以将其改成这样： 

<div class="portal-ui-win">
	<div class="win-title"></div>
	<div class="win-main"></div>
</div>

通用的class命名

大部分时候，我们会使用到通用的名字，比如清除浮动的clear，还有更多的more等等，下面罗列出常用的一些通用命名(持续补充)，请遵循下面的方法。

作用 class
清浮动 clear
更多 more
减少 less
醒目 error
左浮动 fl
右浮动 fr
宽度 w100, w200

书写格式

一行写一个样式，一个样式里的多个属性都在同一行内，模块与模块之间用空行隔开。如下： 

body {font-family: Arial;}

.portal-ui-win {position:absolute;z-index:100;}
.win-title {font-size:14px;line-height:40px;height:40px;}

/*当然，有些时候注释也是少不了的*/
.portal-ui-form {overflow: auto;}
.portal-ui-form label {font-weight: bold;}
.portal-ui-form input[type=text] {padding: 0 5px;}
